# EDA Schematic Editor — AGENTS.md

## Быстрый старт
Менеджера зависимостей (`requirements.txt` / `pyproject.toml`) пока нет. Предполагается, что PySide6 установлен глобально.

```bash
python3 main.py              # запуск приложения
```

## Архитектура
- `main.py` — точка входа: создаёт `QApplication`, инстанцирует `MainWindow`
- `app/main_window.py` — `QMainWindow` с `SchematicCanvas` в качестве центрального виджета, строка состояния показывает привязанные координаты
- `app/canvas.py` — `SchematicCanvas(QGraphicsView)`: кастомная система координат, зум, сетка, панорамирование ПКМ
- `app/items/` — QGraphicsItem для компонентов (`component_item.py`), надписей (`label_item.py`), точек соединения (`junction_item.py`), константы цветов (`colors.py`)

## Особенности холста
- **Система координат**: Y вверх, начало (0,0), единицы — **mil** (1/1000 дюйма). Реализована через `QTransform` scale + отражение Y (`t.scale(zoom, -zoom)`)
- **Сетка**: шаг 100 mil, цвет `#2a2a2a` на фоне `#1e1e1e`. Авторазрежение, когда линии < 5 px друг от друга
- **Зум**: колёсико мыши, множитель 1.1ⁿ, ограничен [0.01, 100.0]. Сохраняет точку под курсором через компенсацию полосами прокрутки после `_update_transform()`
- **Панорамирование**: только ПКМ. Режим `NoDrag`, ручное управление полосами прокрутки. Полосы скрыты
- **Привязка (snap)**: `round(pos / 100) * 100`. Испускает сигнал `position_changed(x, y)` в строку состояния
- **Особенности pen width**: `drawBackground` использует толщину пера **0.0** (косметическое, всегда 1 px независимо от зума). Не используйте целое `1` для косметических перьев в координатах сцены — используйте `0.0`
- **Scene rect**: `(-10000, -10000, 20000, 20000)` в mil. Вид автоматически центрируется на (0, 0) при запуске

## Отрисовка дуг (arcs)
- `SymArc` хранит `x, y, radius, start_angle, sweep_angle` (градусы, CCW от 3 часов — gEDA convention)
- `A`-токен в .sym: ≥6 числовых частей → парсится как `SymArc`; иначе → атрибут
- `paint()`: `drawArc(rect, int(round(start_angle * 16)), int(round(sweep_angle * 16)))` — **углы НЕ инвертируются**.
  - gEDA использует CCW-positive, 0° = 3 часа (Y‑up)
  - Qt `drawArc` использует CCW-positive, 0° = 3 часа (математическая конвенция, независимо от Y)
  - View transform `scale(zoom, -zoom)` зеркалирует Y, но не меняет направление обхода (CCW остаётся CCW на экране)
  - Следовательно, стартовый угол и sweep передаются как есть, без отрицания
- `ComponentGraphicsItem` — оба состояния (выделен/норма) рисуют дуги через `drawArc`
- `library.py`: дуги не распаковываются отдельно, передаются через `sym_data` (`SymData.arcs`)

## Поворот и отражение компонентов и надписей
- Выделить ЛКМ/rubber band, затем **E** (вход в edit mode) + **R** (rotate)
- `ComponentGraphicsItem.rotate(angle_delta)` → `setRotation(self.rotation() + angle_delta)`
- Вращение вокруг центра компонента (0,0 в item-координатах)
- Направление: `-90°` = CW на экране (через Y-flip в view transform)
- Любая клавиша кроме R выходит из edit mode; клик по пустому месту снимает выделение и сбрасывает edit mode
- Поворот применяется ко **всем** выделенным элементам (компоненты и надписи)
- **Ctrl+H** — горизонтальное отражение (`t.scale(-1, 1)`) всех выделенных элементов
- **Ctrl+V** — вертикальное отражение (`t.scale(1, -1)`) всех выделенных элементов
- Отражение комбинируется с поворотом через `item.transform()`; после отражения — `_align_pins_to_grid` для компонентов
- При отражении компонента его дочерние надписи (`LabelItem`) также получают контр-отражение (`scale(-1,1)` или `scale(1,-1)`), чтобы текст оставался читаемым
- При независимом отражении надписи (без родительского компонента в выделении) её собственный `setTransform` отражается

## Выравнивание пинов по сетке
- При размещении и перетаскивании компонент сдвигается так, чтобы точки подключения пинов (`p.x1, p.y1`) оказались в узлах сетки (кратны 100 mil)
- `SchematicCanvas._pin_grid_offset(sym_data, rotation)` вычисляет смещение с учётом поворота: берёт первый пин, поворачивает его вокруг центра компонента на угол `rotation`, возвращает `(-rotated_x % 100, -rotated_y % 100)`
- При `rotation = 0` эквивалентно старой формуле `(cx % 100, (-cy) % 100)`
- `place_component` передаёт `rotation=0`; drag-логика — `item.rotation()`
- После поворота (`_rotate_selected`) вызывается `_align_pins_to_grid`

## Парсинг .sym (токен T)
- `T`-токен: `T x y color size visibility show_name_value angle alignment num_lines` + следующая строка(и) = текст
- `T` внутри блока `P { }` — атрибуты пина (пропускаются)
- `T` вне блоков — текст компонента (refdes, value, device name и т.д.)
- Текст с `key=value` парсится как атрибут и добавляется в `data.attributes`
- `data.texts` хранит `SymText(x, y, content, visible)` — видимые/скрытые тексты
- Скрытые тексты (`visibility=0`) не создают LabelItem, но атрибуты сохраняются

## Надписи (LabelItem)
- `LabelItem(QGraphicsItem)` — дочерний элемент `ComponentGraphicsItem`, позиция и поворот наследуются от родителя
- `_create_labels()` создаёт LabelItem только для `refdes` и `value` из `.sym`:
  - `refdes=R?` → отображается актуальный refdes (например "R1") над компонентом
  - `value=1k` → отображается номинал под refdes (тоже над компонентом)
  - Прочие тексты (например "LM555") игнорируются
- **Клик по надписи** — выделяет только надпись (компонент снимается с выделения)
- **Перетаскивание надписи** — ЛКМ на надписи и перетаскивание (без привязки к сетке)
- **E+R** — поворот всех выделенных: компонент → `setRotation` + `_align_pins_to_grid`; надпись → `setRotation` (независимо от компонента)
- **Delete/Backspace** — удаляет надпись (скрывает `setVisible(False)`)
- **Двойной клик ЛКМ** по компоненту — диалог редактирования имени (refdes) и номинала (value)
- **Двойной клик ЛКМ** по надписи — диалог редактирования текста
- Обновление refdes/value обновляет текст всех LabelItem, чей исходный контент из .sym начинается с `refdes=` или `value=`

## Мультивыделение
- `_selected_items: list[QGraphicsItem]` — все выделенные элементы (ComponentGraphicsItem и LabelItem)
- **Одиночный клик** ЛКМ — выделяет один элемент, снимая предыдущее
- **Клик по пустому** — снимает всё выделение
- Все выделенные компоненты участвуют в перетаскивании (сохраняют взаимное расположение); надписи следуют за родительским компонентом автоматически
- **Delete/Backspace** — удаляет выделенные компоненты со сцены; скрывает выделенные надписи
- Поворот (`E+R`) — все выделенные элементы, у компонентов — `_align_pins_to_grid`
- **Клик ЛКМ** по уже выделенному (в т.ч. после rubber band) не сбрасывает выделение; все выделенные участвуют в drag
- **Ctrl+A** — выделить все элементы на сцене (компоненты, надписи, провода)
- Групповое перетаскивание: если выделены провода вместе с компонентами, они перемещаются совместно; при отрыве от провода пины показываются и на компоненте, и на проводе; при возврате (после отпускания) скрываются заново

## Размещение компонентов (фантом)
- Меню **Добавить → Компонент** — открывает браузер компонентов
- После выбора компонента в диалоге он не ставится сразу, а появляется **фантом** (полупрозрачный) под курсором
- Фантом двигается за указателем с привязкой к сетке 100 mil
- **ЛКМ** — разместить компонент в текущей позиции курсора
- **ПКМ / Esc** — отменить размещение
- `SchematicCanvas.start_placement(sym_data, refdes, value)` — вход в режим
- `SchematicCanvas._cancel_placement()` — выход из режима (удаляет фантом)

## Генерация refdes
- Префикс refdes извлекается из `.sym`: `refdes=R?` → префикс `R`
- Счётчик ведётся по префиксу (`self._refdes_counter` в `MainWindow`)
- Если в `.sym` нет `refdes=?` → fallback на первую букву `sym.id`

## Трассировка проводов (Manhattan)
- **W** — включение/выключение режима проводов; `Escape` — выход из режима
- **ЛКМ по пину** в режиме проводов — начать трассировку
- **ЛКМ по пустому месту** во время трассировки — зафиксировать вершину (ортогональный излом)
- **ЛКМ по другому пину** — завершить провод
- **ПКМ / Escape** во время трассировки — отменить текущий маршрут
- `ManhattanRouter` (`core/router/manhattan_router.py`) — конечный автомат: `IDLE → ROUTING → COMPLETED`
  - Привязка к сетке 100 mil (`round(v / 100) * 100`)
  - Axis-locking: первый сегмент идёт по оси с большим отклонением
  - `process_move(x, y)` → вычисляет `pending_end` с учётом текущей оси
  - `process_click` не нужен — управление напрямую из `canvas.py`
- `WireItem` (`core/router/wire_item.py`) — `QGraphicsItem`:
  - Рисует ломаную (ортогональные сегменты) синим `#4a9eff`, выделение — жёлтым
  - `shape()` с `QPainterPathStroker` для хит-теста (30 mil допуск)
  - Z-value -1 (под компонентами)
- `canvas.py`: состояние `_wire_mode`, методы `_find_pin_at`, `_update_routing_preview`, `_clear_routing_preview`, `_complete_wire`
- `ComponentGraphicsItem.hit_test_pin(scene_pos, tolerance)` — определяет, попал ли курсор в пин
- При удалении компонента связанные с ним провода **не удаляются** (пока)
- Провода выделяются rubber-band и удаляются `Delete`/`Backspace`

## Соединение проводов с компонентами (pin connectivity)
- `_comp_wire_links: dict[tuple[int, int], tuple[WireItem, int, float, float]]` — реестр соединений:
  - ключ `(id(component), pin_index)`, значение `(wire, wire_endpoint_idx, px, py)`
  - Заполняется в `_update_comp_wire_connections(component)` при отпускании
- `ComponentGraphicsItem._connected_pins: set[int]` — индексы пинов, скрытых на компоненте
  - `_update_live_pins(component)` — обновляет только `_connected_pins` при перетаскивании (не трогает `_comp_wire_links`)
- **Три пути установки соединения:**
  1. **Рисование сегмента (N)** — `_fix_segment` → `_update_connections_for_wire(wire)` (мгновенно)
  2. **Pin drag** (перетаскивание конца провода) — `_update_live_pins` при каждом move, `_update_connections_for_wire` на release
  3. **Wire group drag** (перетаскивание целого провода) — `_update_live_pins` + `_refresh_wire_endpoint_pins` при каждом move, `_update_connections_for_wire` на release
- `_refresh_wire_endpoint_pins(scene_pt)` — скрывает/показывает пин на конце провода в зависимости от наличия пина компонента
- `_update_connections_for_wire(wire)` — проверяет оба конца провода, вызывает `_update_comp_wire_connections` для найденных компонентов
- `_break_comp_wire_connections(component)` — разрывает все связи, восстанавливает видимость пинов на проводах
- При старте перетаскивания компонента все `_connected_pins` очищаются, провода открепляются (`_drag_comp_wire_links`); при отпускании связи переустанавливаются
- При старте pin drag или wire drag старые записи в `_comp_wire_links` для этих проводов очищаются, пины на проводах восстанавливаются

## Junction (точки соединения проводов)
- **W** — режим трассировки: клик по пину компонента → начать маршрут
- **N** — режим сегментов: ЛКМ ставит вершины; тоже создаёт junction при попадании на тело провода
- Во время трассировки при наведении на тело существующего провода появляется **белое кольцо** (индикатор) в обоих режимах
- **ЛКМ** по индикатору:
  - Рассекает **все** провода, проходящие через точку соединения
  - Завершает текущий маршрут в этой точке
  - Регистрирует все провода в `WireGraph` (скрывает пины на общей точке)
  - Рисует жёлтый `_JunctionItem` (залитый кружок 20 mil)
- **N-mode** junction: после `_fix_segment(sx, sy)` проверяет `_find_wire_body_at(end_pt, 5.0)` → `_split_wire_at` + `_router.reset()` + `_JunctionItem`
- **W-mode** junction: mouseMoveEvent устанавливает `_wire_hover_pos`, mousePressEvent вызывает `_place_junction(pos)`
- **ПКМ / Escape** — сброс маршрута (можно начать новый)
- `_find_wire_body_at(scene_pos, tolerance)` — определяет, находится ли точка на теле провода (не на конце)
- `_point_on_manhattan_segment(pt, a, b, tolerance)` — проверка точки на ортогональном отрезке
- `_split_wire_at(snapped)` — рассекает ВСЕ провода, проходящие через snapped (кроме концов); возвращает True, если хоть один рассечён
- `_place_junction(pos)` — создание junction (W-mode): рассечение провода через `_split_wire_at`, завершение маршрута через `_router.complete`, визуальный элемент
- `ManhattanRouter.commit(x, y)` — добавить промежуточную вершину, продолжить маршрут
- `ManhattanRouter.complete(x, y)` — завершить маршрут в точной точке, вернуть все вершины
- `_JunctionItem(QGraphicsItem)` — жёлтый кружок z=0
  - Выделяется через rubber-band, Ctrl+A и клик ЛКМ
  - Перемещается вместе с группой выделенных проводов/компонентов
  - Удаляется Delete/Backspace
  - Игнорирует поворот (rotate — no-op)

## Net list экспорт
- **Файл → Экспорт netlist...** — сохраняет `.net` файл в формате gEDA
- `SchematicCanvas.export_netlist()` — группирует провода по связности (WireGraph BFS) и собирает подключения компонентов из `_comp_wire_links`
- Формат: `$NETLIST / $NET имя / refdes pin / $ENDNET / $ENDNETLIST`

## SPICE netlist экспорт (.cir)
- **Файл → Экспорт SPICE netlist...** — сохраняет `.cir` файл для ngspice
- `SchematicCanvas.export_cir()` — группирует провода по связности, назначает имена сетям, генерирует SPICE-строки для каждого компонента
- Поддерживаемые устройства:
  - **R** — `RESISTOR` (2 вывода, номинал из `value`)
  - **C** — `CAPACITOR`, `POLARIZED_CAPACITOR` (2 вывода, номинал)
  - **L** — `INDUCTOR` (2 вывода, номинал)
  - **D** — `DIODE`, `ZENER_DIODE`, `LED` (анод=pin1, катод=pin2, модель из `value`)
  - **Q** — `NPN_TRANSISTOR`, `PNP_TRANSISTOR` (C/B/E по `pinnumber`, модель из `value`)
  - **M** — `NMOS_TRANSISTOR`, `PMOS_TRANSISTOR` (D/G/S по `pinnumber`, модель из `value`, bulk=0)
  - **V** — `VOLTAGE_SOURCE`, `VAC`, `VDC`, `VSIN`, `VPULSE`, `VPWL`, `VEXP`
  - **I** — `CURRENT_SOURCE`
  - **X** — сложные IC (OPAMP, LM555, LM741 и др.) — subcircuit call с `NC` для неподключённых выводов
- **Power символы** (GND, VCC, VDD и т.д. с атрибутом `net=LABEL:PIN`) — не генерируют SPICE-строку, но именуют сеть: `net=GND:1` → узел `0`, `net=Vcc:1` → узел `VCC`
- **Имена сетей**: для unnamed сетей — `N$0001`, `N$0002`, ...
- Неподдерживаемые устройства выводятся как комментарии `* R1: device=... не поддерживается`
- Имена узлов: `NC` для неподключённых выводов в X-компонентах

## Удаление компонентов
- При удалении компонента (`Delete`/`Backspace`) **все** провода, подключённые к его пинам, тоже удаляются
- Связанные junction на концах удалённых проводов пропадают

## Перетаскивание группы (group drag)

### Фазы перемещения в mouseMoveEvent
- **Phase 1 (undo junction)**: перед translate проверяются концы проводов на наличие junction, которого нет в `_drag_junctions`. Если найден:
  - **Обе половины split'а в группе** (`_drag_wires` / `_drag_group_wires`) → junction скрывается (`_hide_junction`) и **добавляется в `_drag_junctions`** чтобы двигаться с группой
  - **Хотя бы одной половины нет в группе** → split отменяется: `_junction_split_map.pop(jitem)`, половинки удаляются из сцены, создаётся сращенный провод (`WireItem` с точками `half_a[:-1] + half_b`), который заменяет половинки в списке перетаскиваемых, если они там были. Junction удаляется из сцены
- **Phase 2 (translate)**: сначала `j.setPos` для всех `_drag_junctions`, затем `w.translate(dx, dy)` + `_refresh_wire_endpoint_pins` для каждого конца каждого провода

### Release
- После создания T-junction (через `_split_wire_at`) — `_refresh_wire_endpoint_pins` для всех концов всех перетаскиваемых проводов, чтобы скрыть пины в точках где есть junction

### Pin drag
- При отрыве конца провода от junction — split отменяется (сращивание половинок), junction удаляется со сцены

### Важно: очистка `_wire_graph` при undo split
- При отмене split'а (сращивание половинок) нужно **обязательно** вызывать `self._wire_graph.remove_wire(half_a/b)` **до** `self._scene.removeItem()`. Иначе старые половины остаются в графе, и при последующем `_split_wire_at` (в release или другом drag) `WireGraph.add_wire` видит их → ошибочно прячет пины на новых половинах (думает, что в точке несколько проводов)
