# SpiceEDA — AGENTS.md

## Быстрый старт
```bash
python3 main.py                           # запуск приложения
python3 -m pytest                         # все 53 теста
python3 -m pytest tests/test_foo.py -v    # один файл
python3 -m pytest -k "test_name" -v       # по имени
pip install -r requirements.txt           # зависимости: PySide6, matplotlib, numpy
```

Нет сборки, нет линтера, нет typecheck, нет CI. Системная зависимость: `ngspice` (установка: `apt install ngspice`).

## Точки входа и структура
- `main.py` — единственная точка входа. Создаёт `QApplication` + `SpiceEDAMainWindow`.
- Единый `UnifiedTabWidget` (QTabWidget) для обеих типов вкладок: `.sch` и `.cir/.sp`.
- Вкладки не закрывают друг друга — каждый файл открывается новой вкладкой справа.
- Переключение между вкладками — кликом по названию (показывает имя файла).
- Вкладки схем авто-экспортируют `.cir` рядом с `.sch` при каждом сохранении.
- **Нет проектной системы** — чисто файловая модель. Нет `.prj` файлов. Только `.sch`, `.cir` на диске.

## Архитектура UI

- **Файл** — всегда видно. Новый .sch (Ctrl+N), Новый .cir (Ctrl+Shift+N), Новая схема (Ctrl+Shift+E), Открыть (Ctrl+O), Сохранить (Ctrl+S), Сохранить как (Ctrl+Shift+S), Сохранить схему в .pdf, Выход (Ctrl+Q).

- **Схема** — видно только при открытых .sch. Новая схема, Добавить компонент (Ctrl+K), Добавить директиву (.), Добавить метку узла (L), Добавить текст (T), Экспорт gEDA netlist, Экспорт SPICE netlist, Просмотр SPICE netlist (Ctrl+Shift+V).

## Редактор схем (EDA/)

Полная документация — в `EDA/AGENTS.md`. Ключевые моменты:
- `SchematicCanvas(QGraphicsView)` — 2062 строки, ядро редактора.
- Система координат: Y вверх, **mil** (1/1000 дюйма), сетка 100 mil.
- Трансформация: `QTransform.scale(zoom, -zoom)` — Y отражён во view.
- Толщина косметического пера: **0.0** (всегда 1 px независимо от зума). Никогда не используйте целое `1` для косметических перьев в координатах сцены.
- Scene rect: `(-10000, -10000, 20000, 20000)` mil.
- Snap: `round(pos / 100) * 100`.
- Панорамирование: только ПКМ. Режим NoDrag, скрытые полосы прокрутки.
- Зум: колесо мыши, множитель 1.1^n, ограничен [0.01, 100.0], сохраняет точку под курсором.
- Начальный зум: `0.25`.

### Клавиши быстрого доступа
| Клавиша | Действие |
|---------|----------|
| **N** | Режим сегментов (свободные ортогональные сегменты) |

| **T** | Добавить текст на схему |
| **L** | Добавить метку узла |
| **Ctrl+H** | Горизонтальное отражение |
| **Ctrl+V** | Вертикальное отражение |
| **Ctrl+A** | Выделить все |
| **Delete/Backspace** | Удалить выделенное (компоненты + прикреплённые провода) |
| **Двойной клик** по компоненту | Диалог изменения refdes/value |
| **Двойной клик** по проводу | Диалог задания цвета |
| **Двойной клик** по метке | Редактирование текста метки |
| **R** | Быстрое размещение резистора (resistor-2) |
| **C** | Быстрое размещение конденсатора (capacitor-1) |
| **D** | Быстрое размещение диода (diode-1) |
| **Q** | Быстрое размещение NPN транзистора (npn-1) |

### Размещение компонентов
1. Перетащить с левой панели, или меню → Добавить → Компонент → браузер.
2. Фантом (полупрозрачный) следует за курсором с привязкой к сетке.
3. ЛКМ — разместить, ПКМ/Esc — отмена.

### Прокладка проводов (Manhattan)
- Только горизонтальные + вертикальные сегменты. Блокировка оси: первый сегмент идёт по оси с большим отклонением.
- Точка излома между двумя сегментами. R переключает HV ↔ VH.
- Привязка к сетке 100 mil.
- Junction (белое кольцо-индикатор) появляется при наведении на тело существующего провода.
- Клик по индикатору: рассекает все провода в этой точке, завершает маршрут, рисует жёлтую точку.
- N-режим: ЛКМ ставит вершины, создаёт junction на теле провода.
- W-режим: клик по пину → начать, клик по другому пину → завершить.

### Соединение проводов
- `_comp_wire_links: dict[(id(component), pin_index), (WireItem, endpoint_idx, px, py)]`
- `_connected_pins: set[int]` на каждом компоненте — скрытые пины.
- Три пути соединения: рисование сегмента (N), перетаскивание пина, перетаскивание группы проводов.
- При удалении компонента **все прикреплённые провода тоже удаляются**.
- Перетаскивание группы: Фаза 1 — обработка junction (split/merge), Фаза 2 — сдвиг.

### Сериализация
- `.sch` файлы — JSON. Формат: `{"version": 2, "format": "spiceeda-schematic", "components": [...], "wires": [...], "junctions": [...]}`.
- `SchematicCanvas.save_sch(filepath)` / `load_sch(filepath)`.
- Вкладки имеют `filepath`, `_dirty`, метка `*` для несохранённых изменений.
- `modified = Signal()` на холсте — испускается при любом изменении (размещение, удаление, провод, поворот, отражение, редактирование двойным кликом).

### Экспорт SPICE из схемы (.cir)
- `SchematicCanvas.export_cir()` — группирует провода по связности (WireGraph BFS), назначает имена цепям, генерирует SPICE-строки.
- Имена цепей: безымянные → `N$0001`, `N$0002`, … (простая нумерация).
- Символы питания (GND, VCC с `net=LABEL:PIN`) — не создают SPICE-строку, только именуют цепь: `net=GND:1` → узел `0`, `net=Vcc:1` → узел `VCC`.
- Поддерживаемые устройства: R, C, L, D, Q (C/B/E по pinnumber), M (D/G/S, bulk=0), V, I, X (subcircuit, NC для неподключённых).
- Неподдерживаемые: выводятся как `* R1: device=... не поддерживается`.
- MODEL и DIRECTIVE компоненты (device=MODEL, device=DIRECTIVE) экспортируются отдельным проходом — их `value` вставляется в нетлист как есть, без привязки к проводам.
- `DirectiveItem` (текстовые директивы, добавленные через меню или ".") экспортируются в порядке обхода сцены как строки SPICE-директив.

### Директивы (DirectiveItem)
- SPICE-директивы на схеме — standalone-тексты (аналог LTspice: `.tran`, `.model`, `.ic`, `.dc`...).
- Добавление: меню **Схема → Добавить директиву…**, горячая клавиша **«.»** (точка).
- Диалог ввода: любая SPICE-директива (с точкой или без).
- `DirectiveItem(QGraphicsItem)` — Z=50 (поверх всего), цвет: `#00ff88` (обычный), `#ffcc00` (выделенный).
- Поведение: выделение кликом / rubber-band / Ctrl+A, перетаскивание (snap к сетке), удаление Delete/Backspace, двойной клик → редактирование текста.
- Сериализация: сохраняются в .sch в секции `"directives"` как `{"text", "x", "y"}`.
- Экспорт: `export_cir()` собирает все `DirectiveItem` со сцены и вставляет строки в нетлист.
- Удаление библиотечной `spice-model-1.sym` не требуется — сосуществуют оба способа.

Роли директорий уже не включает electrical и редактор электрических схем.

## Текстовый редактор SPICE (editor/)

- `QPlainTextEdit` с подсветкой синтаксиса SPICE.
- Цветовые схемы: `SpiceHighlighter.available_schemes()` — 4+ темы.

## Симулятор (simulator/)

### Конвейер (F5)
1. Сохранить текущий `.cir` на диск.
2. `_fix_print_directive()` — убирает пробел перед `(` в `.PRINT`, добавляет тип анализа если отсутствует (напр. `.PRINT` → `.PRINT tran`). Пропускает комментарии.
3. `page.save()` — сбросить содержимое редактора на диск.
4. `validate_netlist(path)` — предварительные проверки:
   - Файл существует и не пустой
   - Директива `.END` присутствует
   - GND (узел 0) подключена
   - Компоненты присутствуют (хотя бы одна строка с SPICE-компонентом)
   - Директива анализа (`.TRAN`, `.DC`, `.AC`, `.OP`) найдена
   - Нет дубликатов refdes
   - Предупреждение если нет `.PRINT`
5. Если валидно: запустить `NGspiceSimulator.run_simulation()` в фоновом потоке.
6. Callback вывода → `_sim_output_queue` → таймер 50ms сбрасывает в терминал.
7. По завершении: авто-открытие `SpicePlotterWindow` если есть данные для графиков.

### Проверки валидатора
- **Ошибки** (блокирующие): пустой файл, нет `.END`, нет GND (узел 0), нет компонентов, нет директивы анализа, дубликаты refdes.
- **Предупреждения** (не блокирующие): нет директивы `.PRINT`.
- `validate_netlist()` возвращает `ValidationResult` с `is_valid`, `errors`, `analysis_type`, `formatted_report()`.

### Особенности ngspice (выстрадано)
- Batch-режим: `ngspice -b file.cir`. Без интерактива.
- `.step param` **не реализован** в ngspice-44. Используйте вложенный `.dc`:
  - Пример: `.dc VCE 0 10 0.1 Ib 0 5m 0.5m` (первый — внутренний sweep, второй — внешний).
- `I(Q1)`, `I(IB)`, `I(V<источник_свипа>)` в `.print` **убивают весь вывод**. Используйте только `I(Vsense)` через отдельный 0V-источник последовательно.
- Трюк измерения: добавьте `Vsense узелA узелB DC 0` последовательно, затем `.PRINT I(Vsense)`.
- Полярность Vsense: `Vsense 0 CE DC 0` (ноль **первым**) → `I(Vsense)` положительный при VCE > VBE (активный режим). `Vsense CE 0 DC 0` даёт отрицательный Ic.
- Направление тока базы NPN: `Ib 0 B DC` (ток **втекает** в базу). `Ib B 0 DC 0` — транзистор закрыт.
- NGspice выводит ток через источник как `vsource#branch` (напр. `vsense#branch`), а НЕ `i(vsense)`. Плоттер распознаёт суффикс `#branch`.
- **Закомментированные директивы не работают**: `* .TRAN 0.1ms 10ms` — ngspice игнорирует их. Не полагайтесь на раскомментирование.
- `.PRINT` должен соответствовать типу анализа: `.PRINT ac` для `.AC`, `.PRINT tran` для `.TRAN`.
- Никогда не ставьте два источника напряжения параллельно (одни и те же узлы) — сингулярная матрица. Используйте один источник для нескольких анализов: `Vin 2 0 SIN(0 1 1k) AC 1`.
- Vsense только **последовательно**, никогда параллельно нагрузке.
- Не добавляйте неподключенные источники питания (Vcc без нагрузки).

### Особенности вывода ngspice
- Вывод TRAN разбивается на несколько таблиц (~50 строк каждая). Если заголовки переменных совпадают — продолжение той же таблицы; если разные — новая таблица/отдельный .PRINT.
- Порядок переменных в заголовке: используйте единую регулярку `re.findall(r'[a-zA-Z][a-zA-Z0-9_]*(?:\([^)]*\)|#[a-zA-Z]+)', ...)` — сохраняет порядок заголовка.
- Детект разорванных таблиц: сброс v-sweep (следующее значение < предыдущего И возврат к начальному). Блоки > 5 точек = отдельный trace.
- Регулярка для детекта данных графиков: `Index\s+(?:time|frequency|v-sweep|i-sweep)`.

## Плоттер (plotter/spice_plotter.py)

- Отдельное окно, тёмный фон, стиль осциллографа.
- Каждая переменная на своём subplot с независимой шкалой Y.
- `subplots_adjust(left=0.12, right=0.98, top=0.95, bottom=0.12)`.
- Толщина линий: **0.5** для линий графиков, 0.8 для курсора/сетки/разделителей. Нет маркеров (501+ точек = каша).
- `hspace=0.05` между subplot (плотное расположение).
- **MultiCursor**: одна вертикальная линия на ВСЕХ осях, горизонтальная линия + информационный текст только на оси под курсором. Blit-рендеринг, 30 FPS.
- Легенда отключена.
- Сохранение в JPEG: Файл → Save plot (Ctrl+S).
- Типы анализа: TRAN (временная область), DC (свип), AC (Боде: дБ, рад), OP (столбцовая диаграмма, напряжения узлов).
- Логика Y-подписей из `_get_ylabel_for_vars()`: `v(` → "Напряжение", `i(` / `#branch` → "Ток", оба → "Напряжение / Ток".
- `SpicePlotterWindow.__init__` принимает `netlist_text: str = ""` — используется DC-парсером для извлечения параметров вложенного свипа.

### DC-плоттер (вложенный sweep)
- `_parse_spice_value(s)` — парсит SPICE-суффиксы (5m, 0.1u, 10k...).
- `_get_outer_sweep_params()` — парсит `.dc` из текста netlist.
- Каждый шаг Ib → отдельный trace `vsense#branch [Ib=X.XmA]`.

### Графика BJT (Lepton EDA style)
- **NPN** (`npn-1`): `device=NPN_TRANSISTOR`, `value=BC547`
- **PNP** (`pnp-1`): `device=PNP_TRANSISTOR`, `value=2N3906`
- Оба имеют корпус в виде окружности (R=316 mil, центр 500,501), две диагональные линии (эмиттер и коллектор), вертикальную линию корпуса и горизонтальную базу
- **NPN**: залитая стрелка **вправо** (от корпуса) — полигон `(510,240)→(601,200)→(555,295)→(535,265)`
- **PNP**: залитая стрелка **влево** (в корпус) — полигон `(600,240)→(509,200)→(555,295)→(575,265)` (зеркальное отражение NPN)
- Пины: C (снизу, tip=600,1000), B (слева, tip=0,500), E (сверху, tip=600,0)
- Все три пина на сетке 100 mil после `_pin_grid_offset` (сдвиг 8 mil по X)

### SPICE-экспорт
Код в `EDA/app/canvas.py:_spice_device_line()` маппит `device=` на SPICE-примитив. Неподдерживаемые device выводятся как `* R1: device=... не поддерживается`.

Компоненты с `device=MODEL` и `device=DIRECTIVE` экспортируются отдельным проходом — их `value` вставляется в нетлист как есть, независимо от подключения проводов (это текстовые аннотации без электрических пинов).

### Привязка пинов
- Пассивные: `pinnumber=1`, `pinnumber=2` — порядок как в SPICE-строке.
- BJT: `pinnumber=C`, `pinnumber=B`, `pinnumber=E` (через `pinnumber_index`).
- MOSFET: `pinnumber=D`, `pinnumber=G`, `pinnumber=S`. 4-й вывод bulk = `0`.
- Power: `net=LABEL:PIN` (напр. `net=GND:1` → узел `0`).
- Opamp (`AOP-Standard`): `pinnumber` → SPICE порядок: 1→N+, 2→N−, 3→V+, 4→V−, 5→OUT. Генерирует `X<refdes> <N+> <N−> <V+> <V−> <OUT> <value>`.

### Генерация refdes
- Счётчик `_refdes_counter[prefix]` в `SchematicTabWidget` (ui/schematic_tabs.py).
- `_on_component_placed` обновляет счётчик после каждого размещения через `max(cur, n)`.
- Префикс извлекается из .sym: `refdes=R?` → префикс `R`, `refdes=A?` → префикс `A`.
- Если в .sym нет `refdes=?` → fallback на первую букву `sym.id`.

#### Полярность VDC/Vsource в экспорте SPICE
- **Проблема**: символ vdc-1.sym имел `+` справа, `-` слева. Но SPICE-экспорт (`canvas.py:2633`) использует `net(0)` как N+ (первый P-блок = левый вывод). В результате при подключении `+` к Vcc и `-` к GND генерировалось `V1 GND Vcc DC 9` → V(Vcc) = −9В.
- **Фикс** (`Sources_Voltage/vdc-1.sym`): перенесён `+` на левый вывод (x=330–370) — туда, где первый P-блок (index 0 = N+). Правый вывод (index 1 = N−) получил `-`.
- **Детали реализации**: экспорт для `VDC`/`VOLTAGE_SOURCE`/`VAC`/`VSIN`/`VPULSE`/`VPWL`/`VEXP`/`CURRENT_SOURCE` использует порядок P-блоков в .sym: `net(0)` = первый P-блок → N+, `net(1)` = второй P-блок → N−. Атрибут `pinnumber` внутри P-блоков НЕ используется для источников — только порядок.
- **Важно**: vsin-1.sym, vpulse-1.sym, idc-1.sym имеют ту же структуру (левый P-блок = index 0 = N+, правый = index 1 = N−), но без визуальной маркировки полярности, поэтому не были изменены.
- **NGspice convention**: `V1 N+ N- DC 9` означает V(N+) − V(N−) = 9В. Для получения V(Vcc)=+9В нужно V1 Vcc 0 DC 9.

#### NGspice особенности (выстрадано)
- **Титульная строка**: первая строка .cir файла ВСЕГДА трактуется как заголовок. Если первой строкой идёт `*` комментарий — он становится заголовком. Если первой строкой идёт компонент (например `V1 2 0 DC 9`) — он НЕ парсится как компонент, а становится заголовком! Всегда начинайте .cir с `*` комментария или `.title`.
- **Ёмкостные переходные процессы**: при больших разделительных конденсаторах (10–100 мкФ) время установления может составлять десятки миллисекунд. Для получения корректного коэффициента усиления по переменному току симулируйте минимум 10–20 постоянных времени.
- **Обратная связь коллектор-база**: усилитель с R_feedback от коллектора к базе имеет усиление ≈ Rc·β/Rf (обычно 1–2). Для высокого усиления используйте делитель напряжения в базе.
- **Коэффициент NE**: в модели BJT параметр NE > 1 (эмиссионный коэффициент) увеличивает эффективное Vt = NE·26mV, что снижает gm и усиление. При NE=1.307 усиление примерно в 1.3 раза ниже, чем при NE=1.

### Рабочий процесс в pcb-rnd
1. **File → Import → Import tEDAx schematics** — импорт `.tdx`
2. **Connects → Subcircuit placement → Disperse all** — разбросать на плату
3. Выделить всё рамкой → **Connects → Subcircuit placement → Auto-place selected subcircuits** — авторазмещение
4. `c` `r` — показать rat lines (связи)
5. **Connects → Optimize routes → Auto-route all** — автотрассировка
